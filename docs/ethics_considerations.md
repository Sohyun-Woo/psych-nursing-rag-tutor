# AI 윤리 고려사항 (AI Ethics Considerations)

본 문서는 본 시스템 설계·운영 과정에서 검토한 AI 윤리 이슈를 기록합니다. 
가천대학교 AI 기본교육과정 사업의 핵심 평가지표인 **"AI 윤리 + 비판적 사고"** 
의 정신간호학적 적용 사례입니다.

---

## 1. 윤리 검토 프레임워크

본 시스템은 다음 윤리 프레임워크를 통합 적용합니다.

| 프레임워크 | 적용 영역 |
|----------|---------|
| WHO Ethics and Governance of AI for Health (2021) | 의료 AI 일반 원칙 |
| EU AI Act High-Risk System 가이드라인 | 교육 AI 위험 관리 |
| ICN 간호사 윤리강령 | 간호 실무 윤리 |
| 한국 「개인정보보호법」 | 데이터 처리 |

---

## 2. 핵심 윤리 이슈와 대응

### 2.1 환각(Hallucination) 위험

**이슈**: LLM은 그럴듯하지만 사실과 다른 응답을 생성할 수 있음. 정신간호 
영역에서 잘못된 정보는 학생 학습뿐 아니라 향후 임상 판단에도 부정적 영향.

**대응**:
- RAG 구조 채택으로 WHO 권위 자료를 인용 근거로 활용
- 모든 응답에 출처(자료명·페이지) 자동 표시
- 시스템 프롬프트에 "근거가 없는 정보는 생성하지 말라" 명시
- 첫 화면에 "AI 피드백은 오류 가능성이 있으므로 교수자와 토론 필요" 고지

### 2.2 알고리즘 편향

**이슈**: LLM의 학습 데이터는 영어·서구 중심. 한국 정신보건 맥락 
(보호의무자 제도, 정신건강복지센터 체계 등)을 반영하지 못할 위험.

**대응**:
- WHO PFA 한국어판 우선 임베딩
- 시스템 프롬프트에 한국 정신보건 체계 맥락 명시
- 학습자에게 "본 시스템의 권고는 국제 표준 기반이며, 한국 임상 현장 적용 시 
  지도교수와의 토론이 필요함" 안내

### 2.3 학습자 자율성과 AI 의존 위험

**이슈**: 학생이 AI 피드백에 과도하게 의존하면 자신의 임상 판단력이 
약화될 수 있음.

**대응**:
- AI는 정답을 단정하지 않고 "근거와 함께 대안 제시" 형태로 응답
- "AI는 한 가지 관점일 뿐이며 교수자·동료와의 토론이 필요함" 명시
- 학습자가 AI 응답에 동의하지 않을 권리 명시
- 세션당 최대 20회로 호출 제한 (남용 방지)

### 2.4 데이터 프라이버시

**이슈**: 학생이 입력하는 응답에 본인의 정신건강 정보가 포함될 가능성.

**대응**:
- 첫 화면에서 "본인의 실제 의료정보·학번·식별정보 입력 금지" 고지
- 영구 저장소 없음 (세션 종료 시 휘발)
- Anthropic API는 입력 데이터를 모델 학습에 사용하지 않음 (정책 확인됨)
- Anthropic API 보관 기간 7일, 이후 자동 삭제 (2025년 9월 정책 업데이트)

### 2.5 위기 상황 오용 위험

**이슈**: 사용자가 본인의 실제 자살 위기를 시스템과 상담하려 할 수 있음.

**대응**:
- 첫 화면에 위기 상황 시 1577-0199 / 1393 안내
- 시스템 프롬프트에 "사용자가 본인의 실제 위기를 표현하면 즉시 전문가 연계 
  안내" 지시
- 응답 화면 하단에 위기 핫라인 정보 상시 표시

### 2.6 환자 사례의 윤리적 처리

**이슈**: 실제 환자 사례를 페르소나로 사용하면 개인정보보호법·정신건강복지법 
위반 가능성.

**대응**:
- **모든 가상 환자 사례는 100% 합성 사례**로 작성
- 본인 임상 8년 경험은 "패턴 지식"으로만 활용, 식별 가능한 사례 사용 금지
- 각 사례 카드 상단에 "본 사례는 교육 목적의 합성 사례이며 실재 환자와 
  무관함" 명시
- 인구학적 정보(나이·성별·직업)는 무작위 재조합

## 3. WHO Ethics and Governance of AI for Health (2021) 6대 원칙 적용

WHO가 2021년 발표한 의료 AI 윤리 가이드의 6대 원칙과 본 시스템의 정렬도:

| WHO 원칙 | 본 시스템 적용 |
|---------|-------------|
| 1. 인간 자율성 보호 (Protect autonomy) | 학생은 AI 피드백을 거부할 권리 보유 |
| 2. 인간의 안녕과 안전 증진 | 환자안전 관점 피드백 + 위기 핫라인 안내 |
| 3. 투명성·설명가능성·이해가능성 | RAG 인용으로 응답 근거 명시 |
| 4. 책임성과 책무성 | 개발자·이용자 책임 범위를 README에 명시 |
| 5. 포용성과 형평성 | 한국어 자료 우선, 접근 장벽 최소화 (무료) |
| 6. 지속가능한 AI 증진 | 오픈소스 공개, CC BY-NC-SA 라이선스 |

## 4. 한국 의료 AI 가이드라인 적용

식품의약품안전처 「의료기기 인공지능 소프트웨어 허가·심사 가이드라인」 
관련 검토:

- **본 시스템은 의료기기가 아님**: 환자 진단·치료·예후 예측을 수행하지 않으며, 
  학생 학습 보조 도구임을 명시
- **임상 의사결정 지원 시스템(CDSS) 해당 여부**: 학생용 학습 도구로, 실제 
  의료진의 임상 의사결정에 사용되지 않으므로 CDSS에 해당하지 않음

## 5. 사용자 고지문 핵심 내용

앱 첫 화면에 표시되는 고지문은 다음 요소를 포함합니다.

1. **시스템 정체성 명시**: 정신간호학 교육용 데모 프로토타입
2. **합성 사례 안내**: 모든 사례는 가상이며 실재 환자와 무관
3. **데이터 처리 투명성**: Anthropic API 경유, 7일 후 삭제, 학습 미사용
4. **입력 금지 정보**: 본인 의료정보, 학번, 환자 식별정보
5. **시스템 한계**: 의료적 진단·치료 도구 아님
6. **위기 상황 연락처**: 1577-0199, 1393

상세 코드 구현은 `src/consent.py` 참조.

## 6. 윤리 검토의 한계

본 윤리 검토는 다음 한계를 가집니다.

1. **단일 개발자 검토**: 다학제 윤리위원회 검토를 거치지 않음 (향후 IRB 신청 
   시 보완 예정)
2. **사용자 검증 부재**: 실제 학생 사용자로부터의 피드백 미수집 (Phase 4에서 
   IRB 승인 후 수집 예정)
3. **법적 자문 부재**: CC 라이선스·개인정보보호법 해석은 일반적 이해에 기반하며 
   법률 자문이 아님
