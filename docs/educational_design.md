# 교육공학적 설계 근거 (Educational Design Rationale)

본 문서는 시스템의 교육공학적 설계 의사결정을 상세히 기록합니다. 
임용 평가 및 향후 효과 검증 연구의 근거 자료로 활용됩니다.

---

## 1. 교육 패러다임

본 시스템은 **구성주의(Constructivism)** 학습 패러다임에 기반합니다. 
지식은 학습자에게 전달되는 것이 아니라, 학습자의 능동적 활동을 통해 
구성된다는 입장입니다.

### 행동주의·인지주의와의 대비

| 패러다임 | 학습관 | 본 시스템의 채택 여부 |
|---------|------|-------------------|
| 행동주의 | 자극-반응의 강화 | ✗ (정답 제시·즉각 보상 모델 아님) |
| 인지주의 | 정보 처리·기억 구조화 | △ (스키마 활성화 일부 활용) |
| **구성주의** | 능동적 의미 구성 | **✓ (핵심 패러다임)** |

## 2. 이론적 기반 상세

### 2.1 Vygotsky의 사회문화이론과 ZPD

Vygotsky(1978)는 **근접발달영역(Zone of Proximal Development, ZPD)** 을 
"학습자가 혼자서는 해결할 수 없으나 더 유능한 타인의 도움으로 해결 가능한 
과제 영역"으로 정의했습니다.

**본 시스템의 적용**:
- AI 튜터는 학생의 ZPD 내에서 **scaffolding(비계)** 역할 수행
- 정답을 직접 제시하지 않고, 학생의 현재 응답에서 출발하여 점진적 개선 유도
- 학생 응답 분석 → 누락 요소 식별 → 근거와 함께 대안 제시의 순환

### 2.2 Peplau의 대인관계이론

Peplau(1952)는 정신간호학의 토대 이론으로, 간호사-대상자 관계의 4단계를 
제시했습니다.

| 단계 | 핵심 과업 | 의사소통 역량 |
|------|---------|-------------|
| **오리엔테이션** | 문제 식별, 신뢰 형성 | **본 시스템 주력 영역** |
| 동일시 | 환자가 간호사에 동일시 | (확장 시 포함) |
| 활용 | 관계 자원 활용 | (확장 시 포함) |
| 종결 | 독립적 기능 회복 | (확장 시 포함) |

본 파일럿은 오리엔테이션 단계의 의사소통에 집중합니다.

### 2.3 Kolb의 경험학습이론

Kolb(1984)의 4단계 학습 순환을 시스템 UX에 직접 구현했습니다.

```
구체적 경험 (Concrete Experience)
   ↓ 학생이 가상 사례에 응답 입력
반성적 관찰 (Reflective Observation)
   ↓ AI 피드백 검토, 누락 요소 인식
추상적 개념화 (Abstract Conceptualization)
   ↓ WHO 권고 원리 학습, 이론 연결
능동적 실험 (Active Experimentation)
   ↓ 재시도, 대안 응답 작성
   ↻ 다음 사이클로 순환
```

### 2.4 Shute의 형성적 피드백 원칙

Shute(2008)는 효과적인 형성적 피드백의 7가지 원칙을 제시했습니다. 
본 시스템은 다음과 같이 적용합니다.

| 원칙 | 본 시스템 적용 |
|------|-------------|
| 구체성 (specific) | "당신의 응답에서 'X 기법'은 사용되었으나 'Y 항목'은 누락되었습니다" |
| 적시성 (timely) | 응답 제출 직후 5초 내 피드백 생성 |
| 준거 참조 (criterion-referenced) | WHO PFA·mhGAP 권고를 명시적 기준으로 사용 |
| 학습자 통제 (learner-controlled) | 학습자가 재시도 시점·횟수 결정 |
| 비교 기준 명시 | 인용된 출처와 페이지 번호 함께 제시 |
| 정교화 (elaborated) | 단순 옳고 그름이 아닌 이유와 대안 제시 |
| 비대조적 (non-comparative) | 다른 학생과 비교하지 않고 개인 학습 궤적에 초점 |

## 3. 학습목표 체계

### Bloom's Taxonomy (Anderson & Krathwohl, 2001 개정판) 매핑

| 수준 | 학습목표 (예시) | 시스템 내 구현 |
|------|------------|-------------|
| **기억 (Remember)** | WHO PFA의 Look-Listen-Link 3원칙을 나열할 수 있다 | 사이드바 학습 가이드 |
| **이해 (Understand)** | 각 원칙의 목적을 자신의 말로 설명할 수 있다 | RAG 피드백 내 원리 설명 |
| **적용 (Apply)** | 자살 시도 환자 사례에 PFA 원칙을 적용한 응답을 작성한다 | 학생 응답 입력 단계 |
| **분석 (Analyze)** | 자신의 응답에 누락된 사정 항목을 식별한다 | 평가 결과 화면 |
| **평가 (Evaluate)** | 자신의 응답을 WHO 권고 기준에 따라 평가한다 | 루브릭 기반 점수표 |
| **창조 (Create)** | 더 효과적인 대안 응답을 구성한다 | 재시도 기능 |

## 6. 평가 루브릭 설계

본 시스템의 형성적 평가는 다음 4개 차원에서 이루어집니다.

### 차원 1: 의사소통 적절성 (PFA 기반)
- 안전감 조성 표현 사용 여부
- 비치료적 의사소통 회피 (비난·평가·조언 등)
- 문화적 민감성 (한국 정신보건 맥락 고려)

### 차원 2: 임상 사정 완전성 (mhGAP 기반)
- 핵심 사정 항목 포함 여부
- 위험요인·보호요인 균형 사정
- 의뢰 기준 인식 여부

### 차원 3: 우선순위 판단
- 안전 위협 우선 대응
- 응급 의뢰 필요성 판단
- 자원 연계 적절성

### 차원 4: 근거 기반 추론
- 의사결정의 근거를 설명할 수 있는가
- WHO 권고와의 일치 여부
- 한계 인식 (AI 피드백의 한계 포함)

## 7. 학습자 인터페이스 설계 원칙

### 인지부하 이론 (Sweller, 1988) 적용
- 한 화면에 하나의 사례만 제시
- 사이드바와 본문 영역 명확히 구분
- 색상 코딩으로 피드백 영역 구분 (의사소통 / 사정 / 대안)

### 자기조절학습 지원
- 학습 진행도 표시 (현재 사례 / 전체 사례)
- 재시도 가능 (실패에 대한 처벌 없음)
- 사이드바에 학습 가이드 상시 제공

## 8. 효과 검증 연구 설계 (향후)

본 시스템의 교육 효과 검증을 위한 연구 설계 초안:

### 연구 가설
- H1: 본 시스템을 활용한 학생군은 통제군 대비 위기개입 지식 점수가 유의하게 높다
- H2: 본 시스템을 활용한 학생군은 통제군 대비 의사소통 자신감이 유의하게 높다
- H3: 본 시스템 사용 시간과 학습 성과 간 양의 상관관계가 존재한다

### 연구 설계
- 유사실험연구 (비동등성 대조군 전후 설계)
- 표본: 정신간호학 실습 수강생 60명 (실험군 30, 대조군 30)
- 측정 도구: 
  - 자체 개발 위기개입 지식 평가지
  - 의사소통 자신감 척도 (검증된 도구 사용)
- IRB 사전 승인 필수
